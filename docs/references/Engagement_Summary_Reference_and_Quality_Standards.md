# Engagement Summary Reference and Quality Standards

## Overview

Output format standards for all agent-generated engagement summary documents.

## Structure Requirements

Each engagement summary must include:

1. **Header** -- Engagement title, date range, stakeholder level
2. **Context** -- Background on the engagement and its strategic relevance
3. **Key Themes** -- 3-5 bullet points capturing the core narrative
4. **Communication History** -- Chronological log of touchpoints
5. **Recommended Next Steps** -- Actionable follow-up items with owners

## Quality Criteria

- **Accuracy** -- All facts must be verifiable against source materials
- **Brevity** -- Follow Smart Brevity rules (see `SMART_BREVITY_RULES.md`)
- **Tone** -- Professional, audience-appropriate, free of jargon
- **Completeness** -- No critical context omitted; unknowns marked as [PLACEHOLDER]

## Formatting Rules

- Use markdown headers for section structure
- Bullet points for lists of 3+ items
- Bold for emphasis sparingly (max 2-3 per section)
- No excessive formatting that reduces readability

## Review Process

All engagement summaries are reviewed before delivery. The agent generates
a draft; the comms lead reviews and edits before sharing with stakeholders.
